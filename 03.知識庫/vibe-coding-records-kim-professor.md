## 背景

金益翰（김익한）：韓國「記錄學者」代表人物，非工程師。
被 NC Soft 邀請對 300 個工程師演講「記錄學者怎麼用 AI」。
演講前三週，用 Claude Code 一天做出個人記錄 app「極致記錄（지독한 기록）」，當場 demo 給工程師看。

## Vibe Coding 定義

> vibe = 振動、感覺、來回對話。Vibe coding = 用嘴巴寫程式，用感覺寫程式，跟 AI 一來一往，code 就慢慢長出來。

## Vibe Coding 五要素（做之前腦袋要有的東西）

| 要素 | 說明 |
|------|------|
| 目的 | 這東西做來幹嘛？給誰用？|
| 輸入 | 使用者要怎麼餵資料進去？|
| 處理流程 | 每一步怎麼動？|
| 判斷標準 | 要做決定的時候，依據什麼？|
| 輸出 | 最後跑出來長什麼樣？|

## 最重要的洞見

> 「記錄學者其實是最適合做 vibe coding 的人，因為我們本來就在練習，把腦袋裡那團模糊的東西寫成具體的步驟。」

真正的門檻不是「會打 code」，是**你能不能把一件事的流程，講清楚、寫下來、想完整**。

工程師能做產品，是因為腦袋習慣這樣思考——不是因為他們會打 code。
田野記錄、課堂筆記、學習歷程的人，一直在練同一件事，只是沒意識到。

## 對應柯維七個習慣

| Vibe Coding 五要素 | 七個習慣 |
|-------------------|---------|
| 目的 | 以終為始（第二個習慣）|
| 處理流程 + 判斷標準 | 要事第一（第三個習慣）|
| 輸出 | 你想活成什麼樣子 |

## 三位一體原則

> 不是「我、AI」兩位一體，是**「我、記錄、AI」三位一體**。

你不寫，AI 就沒有東西可以接住你。
你寫得越具體，AI 給你的回禮就越驚人。
