## 核心比喻

> 大部分人以為在安裝一個工具，但實際上是在請一個 AI 員工進公司。
> 工具裝好就能用。員工需要入職。

「裝好 agent」只解決「它有沒有手腳」，不解決它能不能創造價值。

---

## 三個讓 AI 員工開始有價值的東西

### 1. 世界觀（Worldview）
讓 AI 理解你怎麼看事情，而不是只說「你是一個專業助理」。

最低限度包含：
- 我是誰，我現在在做哪些專案
- 我喜歡什麼溝通方式和風格
- 我的資料邊界和隱私紅線
- 什麼叫做「有用」和「完成」

缺少世界觀 → AI 只能靠猜，猜錯很煩

### 2. Harness（制度/韁繩）
AI 的工作制度，回答四個問題：
1. **什麼不能做**：不能刪資料、不能亂發訊息、不能碰 production
2. **做事前要知道什麼**：先讀規則、查歷史、載入相關 skill
3. **做完後怎麼驗證**：跑測試、貼證據
4. **出錯後怎麼修正**：錯誤變規則，下次自動避開

沒有 Harness 的 AI = 很積極但不知道哪些門不能開的新員工

### 3. 工作方向
不要問「它能做什麼」，要問「你公司缺哪種員工」：

| 你缺的 | AI 的工作 |
|--------|---------|
| 研究助理 | 看新聞、整理趨勢、抓題目 |
| 內容助理 | 維護素材庫、草稿、發布檢查 |
| 工程助理 | 讀 issue、拆任務、跑測試 |
| 營運助理 | 每日提醒、週報、流程檢查 |

## Agent 功能重新定義

| 功能 | 不是… | 而是… |
|------|-------|-------|
| Skills | 收藏技能卡 | 員工的 SOP |
| Memory | 閒聊紀錄 | 員工的長期記憶 |
| Cron | 排程功能 | 員工每天固定上班的節奏 |
| Profiles | 多開帳號 | 不同部門/角色/權限邊界 |

## 入職文件最低版本

```
- 你服務誰：我是誰，最重要的三個目標
- 你怎麼工作：回答風格、查證標準、回報格式
- 你不能做什麼：資料紅線、外部發送、刪改檔案
- 你每天固定做什麼：早報、巡檢、整理
- 怎樣算完成：要有結果 + 證據 + 下一步
```

先給它一個**每天會發生的小工作**，AI 員工靠小循環養成，不靠一次大型任務。
