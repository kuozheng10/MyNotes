## 核心命題

真正的 transferable skill 不是工具，是「把組織邏輯系統化」的能力。

## 你早就在做「人類版 Agent 設計」

如果你做過以下任何一件事：

| 過去做的 | AI 時代的對應 |
|---------|-------------|
| CRM 分眾與自動化 | Lead scoring agent |
| 客服 SOP 與升級流程 | Customer service workflow + escalation logic |
| 供應鏈 exception handling | Alert agent + human-in-the-loop |
| HR approval flow | Approval automation |
| Sales pipeline 管理 | Pipeline orchestration agent |

**工具會變，底層能力不變。**

## 危險的不是 BA，是只執行不思考的人

AI 時代最先被取代的不是某個職稱，而是說不清楚以下問題的人：

- 為什麼做這件事？
- 問題在哪？
- 流程怎麼設計？
- 如何衡量成效？
- 什麼情況要讓人介入（human-in-the-loop）？

> 「主管叫我做什麼就做什麼」的人，會比 BA 更早被取代。

## BA 的升級路徑

```
分析需求的人
  ↓
設計 AI-enabled workflow 的人
```

不需要會寫程式，但需要能描述：
- 流程的輸入/輸出
- 決策點在哪
- 例外情況怎麼處理
- 什麼時候要人工確認

## 企業真正要買的不是 AI 工具

| 企業說要 | 企業真正要 |
|---------|----------|
| AI 工具 | 更快的決策 |
| 自動化 | 更少的人工作業 |
| 數位轉型 | 更穩定的流程 |
| 智慧化 | 更可複製的營運系統 |

## 結論

越到 AI 落地階段，越需要懂 business、懂流程、懂資料、懂組織的人。

**如果你能把 BA / PM / Ops 的經驗，翻譯成「更快決策、更穩流程、更少人工」的價值，你不是被 AI 取代的人，而是幫公司把 AI 用起來的人。**
