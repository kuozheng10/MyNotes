## 核心論點

> Agent 不是讓你把現有的人換掉，而是讓現有的人，第一次有辦法踏進那些他們過去根本進不去的領域。

問錯問題：「Agent 能不能取代這個人？」
問對問題：「這讓我能做到什麼過去做不到的事？」

**能力地圖思維**：你有五個人，地圖就只有五塊。Agent 讓五個人覆蓋一張過去需要十五個人才撐起來的地圖。

## 三種公司類型的困境與解法

| 類型 | 過去的痛點 | Agent 帶來的解法 |
|------|-----------|----------------|
| 新創（5人）| 核心以外的領域要麼不做、要麼粗糙 | 設計多個專業 Agent，創始人作主顧問判斷方向，Agent 執行 |
| 顧問公司 | 核心能力 ↔ 客戶需求之間永遠有缺口 | 核心能力留人，邊緣技能給 Agent |
| 專案型公司 | 農忙人不夠、農閒人閒置，裁/招循環傷文化 | 彈性執行層：核心精簡，高峰快速擴充 |

三種拘束條件不同，但得到的是一樣的：**組織的規模，不再是能力上限的決定因素。**

## 設計決策的差異

問「省多少成本」→ 建出省錢工具，把現有流程便宜化
問「做到什麼過去做不到的事」→ 建出新的組織能力

這個問題的差異，決定了 Agent 架構的整個設計方向。

## 對派哥的啟示

派哥的現況其實就是「五人新創創始人」模式的個人版：

- 主顧問角色：派哥負責判斷方向、整合、最終決策
- Agent 執行層：cc_processor、sales report、MyNotes SOP、TG 通知

目前的架構**已經是能力地圖擴張，不是省錢工具**。
但有一個缺口：每個 Agent 的「能力邊界」沒有明確定義。

可以做的事：
- 每個 cc_processor adapter 的 AGENTS.md 加上「我能做 / 我不能做」兩個區塊
- 讓 Agent 清楚知道什麼時候應該停下來等派哥決策，而不是自己猜
