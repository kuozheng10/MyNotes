## 案例：Dodonov × Stanley

| 項目 | 內容 |
|------|------|
| 主角 | Dodonov（多多諾夫），29 歲 |
| 背景 | 化學工程系 → 會計顧問 → eBay 高級工程師 |
| 產品 | Stanley — LinkedIn AI 內容工具 |
| 開發方式 | **Vibe Coding**（全程零行手寫程式碼） |
| 開發地點 | 倫敦公寓，關門閉關 14 天 |
| 開發時間 | 6 週 |
| 上線 24hr | 250 付費客戶 |
| Stanley 年收 | ~$1億 TWD（ARR） |
| 整體平台年收 | **~$12.5億 TWD** |

---

## 什麼是 Vibe Coding

> 全程只負責「出一張嘴」——描述功能、介面、商業邏輯，讓 AI 生程式碼。

本人不碰一行程式碼，AI 是 24 小時不休息的執行者。

**Stanley 的功能：**
- 替網紅與企業高管在 LinkedIn 自動發文
- 模仿人類語氣
- 自動爬互動數據
- 分析哪種貼文觸及率高 → 自我優化

---

## 核心洞察

**未來最值錢的技能不是寫碼，是懂得怎麼使喚 AI。**

Dodonov 的 eBay 面試故事：
- 技術考題答得坑坑疤疤，快被刷掉
- 拿出自己用業餘時間做的財務數據網站
- 面試官當場被打臉，拿下工作
→ **作品 > 學歷 > 考題**

---

## 與資深工程師 AI 時代的連結

[[senior-dev-ai-era-harness-complexity]] 提到：
- AI 負責「怎麼做」，人守住「要做什麼」
- Vibe Coding 是這個趨勢的極端版本：完全把「怎麼做」外包給 AI

**差異：**
| 面向 | Vibe Coder | 資深工程師 |
|------|-----------|-----------|
| 產出速度 | 極快 | 較慢 |
| 程式碼品質 | 未知、可能是鬼屋 | 可控 |
| 複雜度管理 | 缺乏 | 核心能力 |
| 規模上限 | 低（溫徹斯特神秘屋風險） | 高 |

→ Vibe Coding 適合 **MVP、SaaS 原型、簡單工具**，不適合複雜系統。

## 可操作洞察

1. **先 Vibe，後鞏固**：用 Vibe Coding 快速驗證市場，付費客戶夠多再重構
2. **LinkedIn 內容工具** 是目前 AI SaaS 中最快回本的賽道之一
3. **閉關 + 時間盒**：14 天強制交付，避免無限打磨
