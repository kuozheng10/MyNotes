# Code-Duo — 讓兩個 AI 互相對質的開發工具

> 作者 80 天密集 AI 開發後的總結，開源工具。

---

## 核心問題：AI 太會講話

AI 的信心 ≠ 能力。它在「問題出在自己身上」時會自以為是地一路做下去，錯了改、改了又錯，燒光 Token 才認錯。

**關鍵觀察**：
- AI 只在「問題明顯不在它身上」時才誠實（環境問題、權限問題）
- 問題出在自己身上時 → 自信硬幹 → 鬼打牆 → 最後說「高估了難度」
- Claude 和 Codex 對同一題的解題路徑明顯不同（各有盲點）

---

## 解法：讓兩個 AI 互相審計

**不是讓它們互相稱讚，是真的讓一邊去挑另一邊的毛病。**

- 一邊卡住時，把同樣問題丟給另一邊 → 常冒出完全不同的解法
- 兩邊當場對質 → 話術難成立，胡說八道的機率大降

---

## Code-Duo 四大功能

| 功能 | 說明 |
|------|------|
| **跨 Session 互讀** | Claude 讀得到 Codex 完整輸出，Codex 也讀得到 Claude 的；互相審計的地基 |
| **即時步驟透明** | AI 跑什麼指令、改哪個檔、讀什麼全部即時顯示，不只給最終答案 |
| **照妖鏡** | 對比 AI 宣稱做了什麼 vs 磁碟實際有沒有變動；「嘴上 5 個動作，磁碟 0 變動」即時警告 |
| **Token 追蹤 + 一鍵清空** | 兩邊各自花費即時顯示；一鍵清空舊上下文，不為用不到的舊脈絡付錢 |

---

## 設計原則

- **訂閱制**：不需帶 API key，用月費訂閱，不被 API 計費二次收割
- **Session 延續**：帶得進過去開發的 session，不從零開始
- **同時雙模型**：Claude + Codex 同一畫面，不用切視窗

> 跨 Session 互讀不代表 AI 會自作主張——它們讀到歸讀到，動手還是要你下指令。主導權在人。

## 適合的人

| 類型 | 痛點 |
|------|------|
| 獨立開發者 | 多專案並行，要保住每個專案脈絡 |
| 被 AI 話術整過的人 | 付了時間金錢換來一句「高估了難度」 |
| 判斷不確定時 | 兩個模型給兩種解法 + 互相挑毛病 |
| 在意花費的人 | 把成本攤在眼前，燒得明明白白 |

## 對派哥的評估

✅ **高度相關**：
- 你本來就同時用 Claude Code + Codex，Code-Duo 把兩個整合進同一畫面
- 「照妖鏡」解決了 AI 說做了但實際沒動的老問題
- Token 透明化符合你的額度優化需求

- 訂閱制，費用未知，裝前先確認收費模式
