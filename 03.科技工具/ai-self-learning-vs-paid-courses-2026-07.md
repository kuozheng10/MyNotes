# AI 已經把自學的門檻，降低到趨近於零

> **一句話**：把買課的錢拿去升級 Claude Max 可能更划算——但真人課程還是有 AI 補不了的價值：幫你踩過的坑、整理好的捷徑。

## 核心論點

- 以前學新領域要買書/找老師/買課程/花幾個月摸索方向
- 現在只要打開 ChatGPT/Claude，用這句 prompt 開場：
  > 「我最近對 XXX 領域/技能有興趣，我想透過它來達成 YYY 目標/解決 ZZZ 問題。但我現在是純新手，請你像一位資深的前輩一樣，幫我規劃好學習與實作計劃，帶我一步步完成這個目標。」
- 抱著「霸道的巨嬰心態」，不懂就問到懂，邊學邊做直到達標
- 作者一年來越來越少買課，甚至做課/教課動力幾乎消失

## 但真人課程沒被取代的價值

1. **踩過的坑 + 累積的經驗 + 整理好的捷徑** — AI 未必能精準提供
2. 作者自己買了 Raymond CH Hou 的 Claude Code 課程，理由：
   - 精準解決當下需求（想學會活用 Claude Code）
   - **1小時內能學到真正需要的東西**，不是塞滿知識體系的大雜燴課

## 作者自己做課的教訓

- 第一堂課把會的行銷知識全塞進去，教學資源/工具太多，學員要投入的時間太多 → **完課率非常低**
- 教訓：內容量 ≠ 價值，學員完成率才是關鍵
- 未來想做的課程形式：**3~5天（最多7天）、聚焦帶完成一個小成果的直播課/工作坊**
  - 例：「帶你建立自己的寫作工作流，完成一篇 500 字深度短文」
- 小成果的好處：學員願意短時間投入高強度學習，快速掌握可馬上應用的技能；短期效果雖小，長期像滾雪球累積

## 對派哥的參考價值

- 呼應派哥自己的工作模式：Claude Code + 各種 skill（save-sop、cangjie-skill 等）本質上就是這篇講的「AI 自學」實踐——不是買課，是直接跟 AI 邊做邊學
- 跟 [[cangjie-skill-distill-2026-07]] 是互補視角：cangjie-skill 是把「別人系統性輸出的方法論」蒸餾成工具；這篇講的是「直接用 AI 當前輩，自己邊問邊做」——兩種學習模式可以並用，前者省時間、後者練手感
- 課程設計的「小成果、短時間、高完成率」原則，如果派哥之後想整理教學內容（例如整理 cc_processor/skill 開發心得分享），可以參考這個結構，而不是一次塞全部細節
