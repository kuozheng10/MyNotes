# 公司官網砍掉 CMS/資料庫，改用 AI 生成靜態網站

> 來源：Will 保哥 Facebook 貼文

---

## 原架構問題

Angular前端 + .NET後端 + 功能完整的MVC後台管理系統：

- 改內容要前後端工程師一起處理
- 版型調整耗時數天到數週
- 完整CI/CD部署流程笨重
- 後台管理系統一年登入次數寥寥可數（幾乎沒人用）

## 新架構

用 GPT-5.6 生成純靜態網站，砍掉：後台管理系統、資料庫、CMS。部署在 Azure Static Web Apps 免費額度內，年成本變 $0。

## 好處

1. 維護效率：AI直接改程式碼，人審核後部署
2. 部署簡化：沒有API故障風險、沒有資料庫維護
3. 成本：$0/年
4. 安全：沒資料庫就沒資料庫攻擊面

## 核心洞察

**減少系統複雜度比增加功能更划算。** 跟 [[ponytail-ai-simplicity-plugin]] 同一個精神——能不寫的程式碼/不存在的系統，就是最好的程式碼/系統。

---

## 派哥問「這可行嗎」——我的判斷

**可行，但有明確的適用邊界，不是萬用解法：**

✅ **適合這樣做的情況**：
- 網站內容變動頻率低（公司介紹、產品頁這種，不是每天在改）
- 沒有真的需要「非工程師自己上後台改內容」這個需求（文章裡也講了，後台一年登入沒幾次，證明這家公司本來就用不到）
- 沒有使用者帳號、交易、即時動態資料這類真的需要後端邏輯的東西

⚠️ **真正的風險/取捨，文章沒講清楚的部分**：
- 拿掉CMS等於把「改內容」這件事的門檻從「行銷/業務自己上後台」提高成「要找工程師（或AI+工程師審核）」——如果這家公司後台真的沒人用，這不是問題；但套用到**任何**公司都對，是過度推廣。如果一家公司的行銷天天要自己改活動頁、改文案，砍掉CMS反而是製造瓶頸，不是省事
- Azure Static Web Apps免費額度是真的（100GB頻寬、自訂網域、SSL都含在免費層），$0成本這點站得住腳，不是行銷話術
- 安全性提升是真的（沒資料庫就沒SQL injection/資料庫外洩風險），但部署憑證、git repo存取權限還是要顧好，不是「什麼都不用管」

**結論**：這是「用AI輔助的靜態網站」這個已經成熟的模式（JAMstack的AI強化版），套用在對的網站類型上完全可行且划算。不建議當成「以後每個網站都該這樣做」的通則，要先問「這個網站的內容變動誰在改、多常改」再決定。

跟派哥自己現有的 investment-dashboard / my-wallet-trip / insurance-tracker 比較：這些都是有真實資料流(SQLite/Notion同步)、需要動態更新的儀表板，不屬於這篇文章講的「低頻率靜態內容」範疇，不適合直接套用這個模式砍掉後端——這篇筆記比較像是「未來如果要做一個純資訊展示型網站（例如個人簡歷頁、活動介紹頁）」時可以直接參考的架構選項。
