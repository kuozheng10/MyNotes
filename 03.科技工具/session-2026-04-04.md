## cc_processor v3.0 OCR Bug Fix

### 問題：HSBC 3月旅人卡 帳單漏掉 13 筆

**症狀：** 2/14 恩主公（行天宮醫療志業財團）$10,638、2/26 新安產險 $5,131 等 13 筆沒進 Notion

**根本原因：** HSBC PDF 第3頁有兩張卡（****4778 和 ****3526）。Vision OCR binary `/tmp/ocr_pdf` 對 ****3526 那段的簽帳日期欄位完全讀不到 → `signing_date = None` → parser 跳過整段。

**修法 1：pdfplumber fallback（`cc_processor/ocr.py`）**
- 加 `_extract_signing_dates_pdfplumber()` — 用 pdfplumber 從 PDF 原始文字層讀簽帳日期
- pdfplumber y 座標比 OCR 偏高 ~0.009（bounding box top vs OCR text center），加 offset 修正
- OCR 找不到 signing_date 時，用 pdfplumber 最近 y 的日期補上
- 效果：41 筆 → 54 筆

**修法 2：dedup 改良（`cc_processor/ocr.py` + `cathay_ocr.py`）**
- 舊 dedup：(日期 + 商家前20字 + 金額) 相同就只保留一筆，誤刪真實重複購票
- 新 dedup：額外比對 OCR row 的 y 座標
  - 同 key + y 差 ≤ 0.007 → OCR 雙讀同一行 → 刪
  - 同 key + y 差 > 0.007 → 真實兩筆 → 保留
- 效果：54 筆 → 58 筆，NT$39,135 → NT$39,662（帳單小計 NT$39,668，差 NT$6 ✅）

**修法 3：驗核機制（`cc_processor/base.py`）**
- 每次 TG 訊息加驗核行
  - ✅ 金額驗證通過（差 ≤ NT$50）
  - ⚠️ 金額不符！（差 > NT$50，可能有明細未讀到）
- HSBC 比對：明細加總 vs「消費/新增款項/循環息/其他費用小計」
- 國泰比對：明細加總 vs「正卡本期消費」

### 手動補 import

- 用 `reimport_hsbc_mar.py` 對 Drive 上的 3月旅人卡 PDF 重新跑 OCR
- Notion 補 import：13 筆新增，41 筆 dedup 跳過
