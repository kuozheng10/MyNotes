## cc_processor v3.0 OCR Bug Fix

### 問題：HSBC 3月旅人卡 帳單漏掉 13 筆

**症狀：** 2/14 恩主公（行天宮醫療志業財團）$10,638、2/26 新安產險 $5,131 等 13 筆沒進 Notion

**根本原因：** HSBC PDF 第3頁有兩張卡（****4778 和 ****3526）。Vision OCR binary `/tmp/ocr_pdf` 對 ****3526 那段的簽帳日期欄位完全讀不到 → `signing_date = None` → parser 跳過整段。

**修法 1：pdfplumber fallback（`cc_processor/ocr.py`）**
- 加 `_extract_signing_dates_pdfplumber()` — 用 pdfplumber 從 PDF 原始文字層讀簽帳日期
- pdfplumber y 座標比 OCR 偏高 ~0.009（bounding box top vs OCR text center），加 offset 修正
- OCR 找不到 signing_date 時，用 pdfplumber 最近 y 的日期補上
- 效果：41 筆 → 54 筆

**修法 2：dedup 改良（`cc_processor/ocr.py` + `cathay_ocr.py`）**
- 舊 dedup：(日期 + 商家前20字 + 金額) 相同就只保留一筆，誤刪真實重複購票
- 新 dedup：額外比對 OCR row 的 y 座標
  - 同 key + y 差 ≤ 0.007 → OCR 雙讀同一行 → 刪
  - 同 key + y 差 > 0.007 → 真實兩筆 → 保留
- 效果：54 筆 → 58 筆，NT$39,135 → NT$39,662（帳單小計 NT$39,668，差 NT$6 ✅）

**修法 3：驗核機制（`cc_processor/base.py`）**
- 每次 TG 訊息加驗核行
  - ✅ 金額驗證通過（差 ≤ NT$50）
  - ⚠️ 金額不符！（差 > NT$50，可能有明細未讀到）
- HSBC 比對：明細加總 vs「消費/新增款項/循環息/其他費用小計」
- 國泰比對：明細加總 vs「正卡本期消費」

### 手動補 import

- 用 `reimport_hsbc_mar.py` 對 Drive 上的 3月旅人卡 PDF 重新跑 OCR
- Notion 補 import：13 筆新增，41 筆 dedup 跳過

## 富邦信用卡 Adapter（下午）

**PR**: kuozheng10/MyClaude-config#3（已 merge）

### 背景

富邦帳單 PDF 不在 Gmail 附件，需登入網頁下載，有 CAPTCHA。
解法：用戶手動存到 Google Drive `MyClaude/富邦帳單/`，說「富邦好了」→ Claude 抓最新 PDF 跑完整 pipeline。

### 新建模組

**`cc_processor/fubon_ocr.py`**
- `_roc_to_iso()` — 台灣民國年轉西元（YYY + 1911）
- `parse_transactions_fubon()` — 座標式 OCR 解析：日期 x=0.04–0.13、描述 x=0.13–0.46、金額 x≥0.88；start marker = 消費明細；y-based dedup
- `extract_statement_total_fubon()` — 抓「本期應繳總額」金額
- `extract_due_date_fubon()` — 抓繳款截止日（Bug fix：繳款截止日 header 與值在不同 Y row，改為掃 header 下方 0–0.04 的值行，x=0.38–0.58）
- `detect_card_name_fubon()` — 抓卡別（正卡/附卡末4碼）

**`cc_processor/fubon.py`**
- `check_pending_email()` — 掃 Gmail 未處理富邦帳單 email → TG alert + Notion 待辦（含下載連結）
- `process_drive_pdf()` — Drive 下載 → Keychain 解密（service: fubon-pdf）→ OCR → Drive 分析上傳 + Notion import + TG 報告

**`cc_processor_main.py`**
- `--fubon-pdf [FILENAME]` — 指定檔名或最新
- `--bank fubon` — 只跑 email check
- 預設全跑時自動加入富邦 email check

### Bug Fix

`extract_due_date_fubon()` 原本回傳 None：
- **根因**：OCR 資料中，`繳款截止日` label 和其值 `115/03/30` 在不同 Y row（差 ~0.01）
- **修法**：改為掃 header row Y 下方 0–0.04 範圍內 x=0.38–0.58 的 token，match ROC date pattern
- **結果**：`2026-03-30` ✅

### 測試結果

- `--fubon-pdf --dry-run` ✅
- 正式跑：1筆 匯入 Notion，Drive 分析上傳 ✅
